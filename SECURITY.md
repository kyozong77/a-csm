# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| v0.1.0 (current) | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in A-CSM, please report it to `zon@rzvn.io`.

Please do not report security vulnerabilities through public GitHub Issues, Discussions, or pull requests.

If you require encrypted follow-up, state that in the initial email and a PGP public key can be provided for the disclosure thread.

If the disclosure contact changes, the current project contact will be listed at [rzvn.io/a-csm](https://rzvn.io/a-csm).

We will acknowledge receipt within 72 hours.

Our handling process is:

1. Confirm
2. Assess
3. Remediate
4. Disclose

## Scope

A-CSM is a conversational safety risk assessment framework.

It is not:

- a medical diagnostic tool
- a psychological assessment tool
- a legal compliance certification
